Shell

该笔记来源于MIT-missing semeter

shell是什么

  • shell 是一种计算机程序

  • 他为用户提供了与操作系统内核进行交互的界面

  • 它是一种文本接入入口，相比于图形交互和语音交互，它的功能更加强大

shell的一些简介

  • shell基于空格执行程序，程序后面是希望调用的参数

  • 可以对参数进行转义，要么使用单引号，双引号，要么使用转移符号 \

  • shell是一个编程环境，在shell中执行命令时，实际上就是在执行一段shell可以解释执行的代码

  • 如果你要求shell执行某个指令但是该指令并不是shell所了解的编程关键字时，那么它会咨询环境变量 $PATH 他会列出shell接到某条指令时，进行程序搜索的路径。

shell的基本操作

  1. 打开shell，第一行是提示符 ，可以看到如下代码，是给出了当前目录或者说所在位置

  2. 然后执行程序 date 和 echo ，可以看到输出结果，它会去咨询$PATH环境变量

  5. 同样的，可以通过 cd 切换目录，并且 . 代表了当前目录， .. 代表了父目录, ~ 代表了主目录， - 代表了会切换到上次的目录，ls 表示列出当前目录下的所有文件

root用户

在Liunx和macOS系统中，root用户类似于Windows中的管理员。它的用户ID是0。 通常情况下，不应该作为root用户进行操作，但是偶尔会需要root的权限。那么此时就需要用到一个命令sudo它可以以root用户来执行一些事情。

值得一提的是#代表以root用户执行，$代表以非root用户执行

/sys中的文件并不是计算机上的文件，而是一些内核参数 ，这些内核参数暴露成文件的样子。可以通过文件来访问这些参数。

备注1. 做练习时遇到了一个文件已经写好但是无法执行的情况，原因是没有相应的权限，此时添加权限的命令

777 在这里代表着了权限设置为777表示：

文件所有者（第一个7）有读（4） + 写（2） + 执行（1）的权限（4 + 2 + 1 = 7）
文件所有者所在组（第二个7）有读（4） + 写（2） + 执行（1）的权限（4 + 2 + 1 = 7）
其他用户（第三个7）有读（4） + 写（2） + 执行（1）的权限（4 + 2 + 1 = 7）

而每个权限位代表着

  • 0：没有权限
  • 1：执行权限
  • 2：写权限
  • 4：读权限

将它们组合在一起可以使用相应的数字表示。为了方便说明，可以使用下面的对应关系：

  • 4：读权限（100）
  • 2：写权限（010）
  • 1：执行权限（001）

所以实际上777会使得该文件全面开放
